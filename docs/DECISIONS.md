# 설계 결정 기록 (ADR)

## D1. 백엔드: Supabase (운영) + 로컬 어댑터 (PoC) — 2026-06-29
- **결정**: 사용자 선택은 Supabase 실시간. 단, 키 없이 즉시 실행 가능한 **로컬 어댑터**(localStorage +
  BroadcastChannel)로 PoC를 먼저 완성하고, 동일 인터페이스로 Supabase 어댑터를 끼울 수 있게 설계.
- **이유**: Supabase 프로젝트/키는 사용자가 생성해야 함. PoC 데모는 무설정으로 돌아가야 함.
- **영향**: `DataStore` 인터페이스 유지 → 운영 전환 시 화면 코드 불변.

## D2. 추첨 방식: 공유 키오스크 — 2026-06-29
- **결정**: 셀러가 한 명씩 나와 관리자 화면(`/kiosk`)에서 터치. 현 오프라인 흐름 유지.
- **이유**: 순서·중복 통제 용이, 현장 진행과 동일.
- **영향**: 추첨은 키오스크에서, 셀러는 `/seller`에서 결과 조회(실시간).

## D3. 관리자 접근: 로그인 — 2026-06-29
- **결정**: `/admin`·`/kiosk`는 로그인. PoC는 단일 비밀번호(`AdminGate`), 운영은 Supabase Auth.
- **이유**: 명단 업로드·추첨 진행·자리 이동은 운영자 전용.

## D4. 향제한 처리 = 허용 풀에서 직접 추첨 — 2026-06-29
- **결정**: 향나는 셀러는 제외 좌석을 뺀 풀에서 균등 추첨. 물리적 "재추첨"과 결과적으로 동일·공정.
  UI에선 회피한 좌석을 "재추첨됨"으로 연출.
- **이유**: 공정성 동일 + 무한 재시도 방지. (`lib/lottery/rules.ts`)

## D5. 배치도 = 엑셀 좌표 정규화 SVG — 2026-06-29
- **결정**: 원본 엑셀 셀 좌표를 추출·정규화해 SVG로 렌더(스키매틱). x축 ×5.25 비율 보정.
- **이유**: 운영팀이 쓰던 배치도와 동일 인식. 두 band(윗/아랫구간) 스택으로 가로폭 활용.

## D6. 디자인: 따뜻한 마켓 + 조각보 컬러 — 2026-06-29
- **결정**: 코랄/테라코타 프라이머리 + 테일 액센트 + 14색 패치워크 카테고리. Pretendard.
- **이유**: 부평구문화재단 CI의 조각보 모티브, 마켓 분위기. (색/로고는 위임받아 직접 선정.)

## 미해결 / 후속
- 셀러 명단에 **전화번호 컬럼 없음** → 검색은 이름/상호. 추후 컬럼 추가 시 전화번호 검색 지원(파서는 이미 대응).
- 기능 6-B(지도 측위)는 현장 기준점 위경도 실측 필요. ([FEATURE6_GPS](./FEATURE6_GPS.md))
- 식품·음료의 향제한 여부는 기본 false. 필요 시 관리자가 셀러별 토글.
